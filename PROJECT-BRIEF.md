## 课题来源

课题六《运维数字员工的建设》，选自《广东公司提供实训课题2026年版》。

## 课题原文

为提升企业运营效率、为一线员工减负、为流程增效，企业采用AI+RPA等技术构建数字员工，通过软件机器人替代或辅助人类员工完成那些重复性高、规则性强或基于数据分析的复杂任务。

在企业运营运维过程中，运维工作重复性高、流程性强，适合通过打造运维数字员工来提高运维效率。简要步骤如下：

**1）大模型本地部署 + RAG私有知识库**
- 可以部署DEEPSEEK等开源大模型在本地服务器。
- 企业知识库因安全考虑一般进行私有化部署，可以部署RAG工具（如anythingLLM等）。
- 构建本地运维知识库，包括运维常见问题处理方案，可以根据实际工作需要进行构建FAQ问答对。例如：
  - 问题：账号冻结怎么处理？
  - 解决方案：通过自助方式访问网址 http://XXX.YYY.ZZZ 或者拨打热电话XXXXX。

**2）应用集成**
- 再搭建知识库后，开发运维申告门户，具备运维人员账号的增删改查等后台维护功能。增就是创建运维账号，设置密码，填写运维人员身份信息；删就是对运维账号进行冻结清理；改就是对账号信息进行修改；查就是提供多种方式对运维账号进行查询。
- 前台提供运维基础维护服务，一是提供各类问题的自助查询服务，调用以上的知识库能力；二是当遇到无法回答问题时，提供在线记录或者转人工服务的功能。
- 后台运维人员定期查询机器难以处理的在线记录，进行人工干预处理，并对报障人进行回访处理，处理完成后再自动完善知识库的内容信息。

**参考资料**：AI、RPA

**硬件要求**：PC机 4C8GB内存 100GB硬盘

**测试环境**：连接互联网

## 业务理解（已确认）

这是一个**企业内部 IT 运维支持系统**。场景如下：

```
一线运维人员（各地市基层工程师）
    │  日常遇到技术问题
    ▼
打开运维门户 → 输入问题 → AI 从知识库检索回答
    │                    │
    │                    ├── 能回答 → 返回答案 ✓
    │                    │
    │                    └── 不能回答 → 生成工单
    │                                      │
    ▼                                      ▼
查看"我的工单"状态              资深运维专家 领取工单
                                          │
                                          ▼
                                    人工研究后给出方案
                                          │
                                          ▼
                                    回访报障人，确认解决
                                          │
                                          ▼
                                   一键录入知识库（下次 AI 就能回答了）

系统管理员：管理账号（入职/离职）、管理知识库内容质量
```

## 三种角色

| 角色 | 他们是谁 | 核心职责 |
|------|----------|----------|
| **一线运维人员** (operator) | 基层运维工程师，在机房/基站/营业厅处理日常运维 | 遇到问题 → 自助搜索答案；搜不到 → 提交工单 |
| **资深运维专家** (expert) | 经验丰富的技术骨干/老师傅 | 处理 AI 搞不定的工单 → 回访确认 → 把方案录入知识库 |
| **系统管理员** (admin) | IT 部门管理者 | 管理账号（入职开账号/离职冻结）、管理知识库 FAQ |

## 核心业务闭环

```
自助查询 → AI 兜底 → 工单流转 → 人工处理 → 回访确认 → 知识入库 → 自助查询（循环）
```

## 已锁定的约束

| # | 约束 | 说明 |
|---|------|------|
| 1 | **不做手机 APP** | 仅 Web 门户 |
| 2 | **不做 Docker** | 仅提供直接启动脚本（start.bat / start.sh） |
| 3 | **单机部署** | 4C8G 100GB 一台 PC |
| 4 | **本地模型** | 大模型必须本地运行，不得调用云端 API |
| 5 | **仅三类角色** | operator / expert / admin，无其他 |
| 6 | **中文环境** | 界面、知识库全部中文 |
