## 1. 引言

### 1.1 编写目的

本文档为"运维数字员工门户系统"的软件需求规格说明书。文档旨在明确系统的功能需求、非功能需求、数据模型和验收标准，为后续的架构设计、详细设计和编码实现提供唯一的、确定的需求基线。

本文档的预期读者包括：项目架构师、后端开发工程师、前端开发工程师、测试工程师和项目管理人员。

### 1.2 适用范围

本文档覆盖"运维数字员工门户系统"的第一阶段交付范围，包括：
- RAG 智能问答子系统
- FAQ 知识库管理子系统
- 工单流转子系统（含回访确认）
- 用户认证与权限管理子系统
- 系统管理后台

本文档不覆盖：移动 APP 端、容器化部署、多机集群部署、云端 API 调用。

## 2. 总体描述

### 2.1 产品背景

企业内部 IT 运维部门的一线运维人员（基层工程师）在日常工作中频繁遇到各类技术问题。传统模式下，问题需要逐级上报或电话咨询资深专家，效率低且重复性高。本系统通过部署本地大模型（DeepSeek）和 RAG 私有知识库，构建一个 Web 门户作为"运维数字员工"，使一线运维人员能够自助查询问题答案。对于 AI 无法回答的问题，系统自动生成工单流转给资深运维专家处理，处理完成后方案一键录入知识库，形成知识闭环。

### 2.2 产品范围

本系统实现以下核心业务闭环：

```
自助查询 → AI 兜底 → 工单流转 → 人工处理 → 回访确认 → 知识入库 → 自助查询（循环）
```

系统边界：
- **包含**：FAQ 知识库管理、RAG 问答引擎、工单流转与处理、用户认证与权限管理、账号管理、仪表盘统计、预置示例数据
- **不包含**：手机 APP、Docker 容器化、多机集群、外部系统集成、在线客服实时聊天

### 2.3 用户角色

| 角色 | 英文标识 | 典型用户 | 核心职责 |
|------|---------|---------|---------|
| 一线运维人员 | operator | 基层运维工程师，在机房/基站/营业厅处理日常运维 | 自助搜索答案；搜不到则提交工单；查看工单处理进度 |
| 资深运维专家 | expert | 经验丰富的技术骨干/老师傅 | 领取 AI 无法处理的工单；研究并给出方案；回访报障人确认；一键将方案录入知识库；浏览 FAQ 知识库辅助判断 |
| 系统管理员 | admin | IT 部门管理者 | 账号的增删改查与冻结/解冻；FAQ 知识的增删改查与批量导入导出；管理知识库内容质量 |

### 2.4 设计约束

| 编号 | 约束 | 说明 |
|------|------|------|
| C-01 | Web 门户 | 仅提供浏览器访问的 Web 系统，不提供手机 APP |
| C-02 | 直接启动 | 不采用 Docker，仅提供 start.bat / start.sh 直接启动脚本 |
| C-03 | 单机部署 | 部署在 4C8G 100GB 硬盘的一台 PC 上 |
| C-04 | 本地大模型 | 大模型必须在本地运行，不得调用云端 API。确定选型：DeepSeek 开源模型（deepseek-llm-7b-chat），通过 Ollama 加载运行 |
| C-05 | 三类角色 | 仅 operator / expert / admin，不引入额外角色 |
| C-06 | 中文环境 | 界面中文，知识库中文。Embedding 模型通过 Ollama 运行 nomic-embed-text（约 274MB，768 维向量），内存占用约 0.3GB |
| C-07 | 向量数据库 | 确定选型：ChromaDB，轻量级、Python 原生、支持单机文件存储、无需独立数据库服务 |
| C-08 | Web 框架 | 后端确定选型：Python FastAPI，异步支持好、与 ML 生态无缝集成、自动生成 OpenAPI 文档 |
| C-09 | 前端框架 | 确定选型：Vue 3 + Element Plus，成熟的中文 UI 组件库、企业级后台管理系统广泛采用 |
| C-10 | 数据持久化 | 确定选型：SQLite，单机部署无需独立数据库服务进程，Python 内置支持 |

## 3. 功能需求

### 3.1 用户认证与权限管理

#### F-1.1 用户登录

**描述**：系统提供登录页面，用户输入账号和密码进行身份认证。验证通过后根据角色跳转至对应的首页。连续 5 次密码错误则临时锁定账号 15 分钟。

**前置条件**：用户账号已创建且未被冻结。

**验收标准**：
1. 登录页面访问 `/login` 路由，显示账号输入框、密码输入框、登录按钮
2. 输入正确的账号密码后，系统在 2 秒内返回登录成功，服务端下发 JWT Token（有效期 30 分钟）
3. 前端根据 Token 中携带的角色信息跳转至对应首页（operator→问答页，expert→工单列表页，admin→管理后台）
4. 输入错误的账号或密码时，页面提示"账号或密码错误"
5. 同一账号连续 5 次输入错误密码，该账号被锁定 15 分钟
6. 已冻结的账号登录时返回"账号已被冻结，请联系管理员"

#### F-1.2 用户登出

**描述**：用户点击登出按钮，系统清除服务端会话状态和客户端 Token，跳转至登录页面。

**前置条件**：用户已登录。

**验收标准**：
1. 登出后客户端 Token 被清除
2. 服务端将该 Token 加入黑名单，无法再次使用
3. 登出后自动跳转到登录页面

#### F-1.3 会话维持与自动刷新

**描述**：系统在用户操作过程中自动维持会话。如用户超过 30 分钟无任何操作，会话过期，自动登出。

**前置条件**：用户已登录。

**验收标准**：
1. 每次前端 API 请求，如会话剩余 < 5 分钟，服务端自动刷新 Token
2. 前端检测到新 Token 后更新本地存储
3. 用户无操作超过 30 分钟，下次请求返回 401，前端自动跳转至登录页
4. 前端实现在会话即将过期前 1 分钟弹出续期提示

#### F-1.4 密码修改

**描述**：已登录用户可以修改自己的密码，修改时需要验证原密码。

**前置条件**：用户已登录。

**验收标准**：
1. 提供密码修改页面，包含"原密码""新密码""确认新密码"三个输入项
2. 新密码必须满足安全策略（长度至少 8 位，包含大写字母、小写字母、数字、特殊字符中的至少 3 类）
3. 原密码验证失败时提示"原密码错误"
4. 新密码与确认新密码不一致时提示"两次输入的密码不一致"
5. 密码修改成功后，强制登出
6. 新密码不能与前 3 次的历史密码相同

#### F-1.5 基于角色的页面访问控制

**描述**：系统根据用户角色控制前端页面访问权限和后端 API 调用权限。

**前置条件**：用户已登录。

**验收标准**：
1. 越权访问 URL 时返回 403 并提示"无权限访问"
2. 前端路由根据角色动态配置，非授权路由不显示在导航菜单中
3. 后端 API 层实现角色鉴权中间件
4. 权限矩阵详见附录 7.1

### 3.2 账号管理（管理员）

#### F-2.1 创建运维账号

**描述**：系统管理员可以创建新的运维账号（operator 或 expert 角色）。

**前置条件**：当前用户角色为 admin。

**验收标准**：
1. 提供账号创建表单，字段：账号名（必填，唯一，3-20 位字母数字下划线）、姓名（必填，2-20 位中文）、手机号（必填，11 位数字）、角色（必填，operator/expert）、邮箱（选填）、所属部门（必填）、备注（选填）
2. 账号名唯一性检查，重复时提示"账号名已存在"
3. 密码由系统自动生成（12 位随机，含大小写字母、数字、特殊字符），明文展示一次，提示"请提醒用户首次登录后修改密码"
4. 创建成功后返回账号列表页
5. 页面响应时间不超过 2 秒

#### F-2.2 账号查询

**描述**：系统管理员可以通过多种方式查询运维账号。

**前置条件**：当前用户角色为 admin。

**验收标准**：
1. 查询条件：账号名（模糊）、姓名（模糊）、手机号（精确）、角色（下拉）、状态（全部/正常/冻结）
2. 所有查询条件可组合使用
3. 分页表格展示，每页默认 20 条
4. 表格列：账号名、姓名、手机号、角色、状态、最后登录时间、创建时间、操作按钮
5. 查询结果为空展示"未找到匹配的账号"
6. 查询响应不超过 3 秒（5000 条数据内）

#### F-2.3 账号信息修改与冻结/解冻

**描述**：系统管理员可以修改账号信息，冻结或解冻账号。

**前置条件**：当前用户角色为 admin。

**验收标准**：
1. 可修改姓名、手机号、邮箱、所属部门、角色、备注。账号名不可修改
2. 冻结操作弹出确认对话框，确认后账号状态变为"冻结"
3. 解冻操作弹出确认对话框，确认后账号状态恢复为"正常"
4. 不可冻结 admin 自身的账号
5. 所有操作记录写入操作日志

#### F-2.4 密码重置

**描述**：系统管理员可以重置指定账号的密码。

**前置条件**：当前用户角色为 admin。

**验收标准**：
1. 弹出确认对话框后系统生成随机密码（12 位）
2. 密码以明文展示一次，提示"请记录此密码并告知用户"
3. 该账号所有活跃会话立即失效

### 3.3 FAQ 知识库管理（管理员）

#### F-3.1 FAQ 新增

**描述**：系统管理员在知识库中新增一条 FAQ 问答对。

**前置条件**：当前用户角色为 admin。

**验收标准**：
1. 表单字段：问题（必填，最多 500 字）、分类（必填，预定义分类）、答案（必填，Markdown，最多 10000 字）、标签（选填，最多 5 个）、关联关键词（选填，最多 10 个）
2. 提交后写入 SQLite，同时触发向量化
3. 向量化完成后可在 RAG 检索中被命中
4. 整个流程（含向量化）响应不超过 15 秒

#### F-3.2 FAQ 修改

**描述**：系统管理员修改已有 FAQ 的内容。

**前置条件**：当前用户角色为 admin。

**验收标准**：
1. 加载该条 FAQ 当前内容至编辑表单
2. 提交后 FAQ 表和向量库同步更新（删旧向量→重新 Embedding→插入新向量）
3. 版本号自增

#### F-3.3 FAQ 删除

**描述**：系统管理员删除一条或多条 FAQ。

**前置条件**：当前用户角色为 admin。

**验收标准**：
1. 提供单条删除和批量删除
2. 删除前弹出确认对话框"此操作不可恢复"
3. 确认后从 FAQ 表和向量库中同时删除

#### F-3.4 FAQ 查询

**描述**：系统管理员通过多种条件查询 FAQ 列表。

**前置条件**：当前用户角色为 admin。

**验收标准**：
1. 查询条件：问题关键字（模糊）、分类（下拉）、状态（已发布/草稿）、创建时间范围
2. 分页表格展示，每页 20 条
3. 表格列：ID、问题摘要（前 50 字）、分类、状态、版本号、创建时间、操作按钮
4. 点击可查看 FAQ 完整详情

#### F-3.5 FAQ 批量导入

**描述**：系统管理员通过 Excel 或 CSV 文件批量导入 FAQ 数据。

**前置条件**：当前用户角色为 admin。

**验收标准**：
1. 支持上传 .xlsx 或 .csv 格式文件（含模板下载功能）
2. 模板格式：.xlsx 表头为"问题""答案""分类""标签""关键词"五列；CSV 文件编码为 UTF-8，表头同上
3. 逐行校验：问题为空则跳过，答案为空则跳过，分类不在预定义列表则跳过
4. 导入完成后显示统计："成功导入 [N] 条，跳过 [M] 条（详见错误记录）"
5. 提供错误记录下载（.txt 格式，标明每行错误原因）
6. 导入的数据自动触发向量化
7. 单次导入最大支持 5000 条

#### F-3.6 FAQ 批量导出

**描述**：系统管理员将 FAQ 数据导出为 Excel 文件。

**前置条件**：当前用户角色为 admin。

**验收标准**：
1. 可按当前查询条件导出全部或选中导出
2. 导出文件格式与导入模板一致
3. 导出文件名 `FAQ_export_YYYYMMDD_HHmmss.xlsx`
4. 10000 条数据以内导出不超过 30 秒

#### F-3.7 FAQ 状态管理

**描述**：FAQ 发布状态（已发布/草稿）。草稿状态不会被 RAG 检索命中。

**前置条件**：当前用户角色为 admin。

**验收标准**：
1. 新增 FAQ 默认状态为"已发布"
2. 可将已发布 FAQ 改为"草稿"
3. 草稿状态在向量库中标记为禁用或在检索时过滤
4. 可将草稿重新发布

### 3.4 RAG 智能问答（一线运维人员）

#### F-4.1 提交问答请求

**描述**：一线运维人员在问答页面输入问题，系统调用 RAG 流程进行检索和生成。

**前置条件**：用户已登录，角色为 operator。

**验收标准**：
1. 问答页面提供文本输入框（多行，最多 2000 字符）和"发送"按钮
2. 提交后页面显示"AI 正在思考..."加载状态
3. 系统在 30 秒内返回回答（含完整 RAG 推理时间）
4. 回答展示在对话区域，标明"AI 回答"或"未找到答案"
5. 整个问答过程记录到问答历史表

#### F-4.2 RAG 检索流程（后台自动执行）

**描述**：系统接收到用户问题后，执行完整的 RAG 检索生成流程。

**前置条件**：F-4.1 发起问答请求。

**验收标准**：
1. **文本预处理**：中文分词、去除停用词、特殊字符过滤
2. **Embedding 向量化**：使用 nomic-embed-text 模型将问题转化为 768 维向量
3. **ChromaDB 检索**：余弦相似度检索 Top-5 最相似 FAQ
4. **相似度阈值判断**：最高相似度 < 0.65 则判定为"无法回答"
5. **LLM 生成回答**：相似度 ≥ 0.65 的检索结果作为上下文，构造 Prompt 送入 DeepSeek 生成回答
6. **降级引导工单**：相似度 < 0.65 时，不调用 LLM，直接返回"无法找到相关答案，是否创建工单？"
7. 全程日志记录：问题、命中 FAQ、相似度分数、回答/降级结果、处理耗时

#### F-4.3 查看问答历史

**描述**：一线运维人员可以查看自己的历史问答记录。

**前置条件**：用户已登录，角色为 operator。

**验收标准**：
1. 问答页面展示当前会话历史（按时间倒序）
2. 每条记录展示：问题、回答/无法回答标识、时间
3. 支持清空当前对话历史（不影响数据库）
4. 提供"查看历史记录"页面，支持按日期范围查询

### 3.5 工单管理（一线运维人员）

#### F-5.1 创建工单

**描述**：当 AI 无法回答时（F-4.2 降级），用户可点击"创建工单"提交工单。用户也可主动创建工单。

**前置条件**：用户已登录，角色为 operator。

**验收标准**：
1. AI 返回"无法回答"界面时显示"创建工单"按钮
2. 工单表单自动填充问题描述（用户上一步输入的问题）
3. 用户可补充：问题描述（可编辑）、补充说明（选填，最多 2000 字）、紧急程度（必填，一般/紧急/特急）
4. 提交后生成工单，状态"待处理"，工单编号自动生成（格式：TK-YYYYMMDD-XXXXX）
5. 创建成功后展示工单详情页

#### F-5.2 查看我的工单

**描述**：一线运维人员查看自己提交的工单列表及处理状态。

**前置条件**：用户已登录，角色为 operator。

**验收标准**：
1. 分页展示当前用户提交的所有工单
2. 列表包含：工单编号、问题摘要、紧急程度、状态、创建时间、处理人
3. 支持按状态筛选（全部/待处理/处理中/待回访确认/已完成）
4. 点击工单编号可查看完整详情

#### F-5.3 工单详情查看与回访确认

**描述**：一线运维人员查看工单详情，并在专家给出方案后进行回访确认。

**前置条件**：用户已登录，角色为 operator。

**验收标准**：
1. 工单详情页展示：工单编号、问题描述、创建时间、紧急程度、当前状态、状态变更时间线
2. 状态为"待回访确认"时，展示"确认解决"和"未解决"两个按钮
3. 点击"确认解决"：工单状态变为"已完成"
4. 点击"未解决"：工单状态回退为"处理中"，备注"报障人反馈未解决，需重新处理"
5. 全部变更记录写入工单日志表

### 3.6 工单处理（资深运维专家）

#### F-6.1 查看待处理工单列表

**描述**：资深运维专家查看"待处理"和"处理中"状态的工单。

**前置条件**：用户已登录，角色为 expert。

**验收标准**：
1. 列表默认展示"待处理"和"处理中"的工单
2. 列表包含：工单编号、问题摘要、紧急程度（紧急/特急高亮）、提交人、提交时间、当前状态
3. 支持按紧急程度和状态筛选，支持关键词搜索
4. 紧急工单置顶，特急工单红色标记

#### F-6.2 领取工单

**描述**：专家领取"待处理"工单。

**前置条件**：用户已登录，角色为 expert，工单状态为"待处理"。

**验收标准**：
1. 工单详情页点击"领取工单"
2. 领取后状态变为"处理中"，处理人设为当前专家
3. 同一工单只能被一个专家领取

#### F-6.3 填写处理方案

**描述**：专家研究问题后在系统中填写处理方案。

**前置条件**：角色为 expert，工单状态为"处理中"且当前用户是处理人。

**验收标准**：
1. 提供"处理方案"编辑区域，支持 Markdown，最多 10000 字
2. 点击"提交方案"后状态变为"待回访确认"
3. 支持"方案暂存"功能，多次编辑保存

#### F-6.4 退回工单

**描述**：专家可以退回不属于自己职责范围或需更多信息的工单。

**前置条件**：角色为 expert，工单状态为"处理中"且当前用户是处理人。

**验收标准**：
1. 提供"退回"按钮，弹出文本框填写退回原因（必填，最多 500 字）
2. 退回后状态变为"待处理"，处理人清空，退回原因记录在工单日志中

#### F-6.5 方案录入知识库

**描述**：一线运维人员确认解决后，专家将本次工单的"问题+方案"一键录入知识库。

**前置条件**：工单状态为"已完成"。

**验收标准**：
1. 已完成工单详情页展示"录入知识库"按钮
2. 弹窗预填充问题和方案内容，可编辑
3. 确认后 FAQ 表新增记录，触发向量化
4. 录入成功提示"方案已录入知识库，下次 AI 可回答相似问题"
5. 同一问题已存在时提示"是否覆盖？"

### 3.7 仪表盘与统计

#### F-7.1 角色仪表盘

**描述**：系统首页为不同角色提供统计概览看板。

**前置条件**：用户已登录。

**验收标准**：
1. **operator 首页**：快速问答输入框 + 我的工单统计（待处理/处理中/已完成数量）
2. **expert 首页**：待处理工单总数、我的工单数（处理中/待回访确认/已完成）、近 7 天工单处理趋势图
3. **admin 首页**：总账号数、活跃账号数、FAQ 总数、今日问答次数、各状态工单数量、近 7 天问答趋势图

### 3.8 系统初始化

#### F-8.1 预置示例数据

**描述**：系统首次启动时自动初始化示例 FAQ 数据和 admin 账号。

**前置条件**：系统首次启动，FAQ 表为空。

**验收标准**：
1. 首次启动自动检查 FAQ 表是否有数据
2. 若为空，自动插入不少于 20 条示例 FAQ，覆盖至少 5 个分类
3. 示例数据同步向量化
4. 自动创建 admin 账号（账号名：admin，初始密码：Admin@123456），提示"首次登录后请修改默认密码"

#### F-8.2 FAQ 变更实时同步向量库

**描述**：FAQ 增删改操作实时同步到 ChromaDB 向量库。

**前置条件**：系统运行中。

**验收标准**：
1. FAQ 新增：写入后立即 Embedding 并插入 ChromaDB
2. FAQ 修改：更新后删除旧向量，重新 Embedding 插入新向量
3. FAQ 删除：删除后立即删除 ChromaDB 对应向量
4. 同步失败：FAQ 表数据保持已保存状态，记录错误日志，自动重试 3 次（间隔 5 秒）

       ↑                          │   │
       │        expert 退回        │   │ 提交方案
       └──────────────────────────┘   │
                                      ▼
                              ┌─────────────────┐
                              │   待回访确认      │
                              │pending_confirm   │
                              └────────┬────────┘
                                       │

                          │ 确认解决   │ 反馈未解决  │
                          ▼            ▼            │
                    ┌──────────┐  ┌──────────┐     │
                    │  已完成   │  │  处理中   │─────┘
