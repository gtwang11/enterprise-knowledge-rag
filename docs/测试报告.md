## 1. 测试概述

### 1.1 测试目标

验证"运维数字员工门户系统"的功能完整性、正确性和稳定性，确保系统满足 SRS 中定义的全部验收标准。

### 1.2 测试范围

| 子系统 | 测试范围 |
|--------|---------|
| 用户认证 | 登录/登出/Token刷新/密码修改/登录锁定/限流 |
| 账号管理 | 账号CRUD/冻结解冻/密码重置/角色鉴权 |
| FAQ 知识库 | FAQ CRUD/批量导入/批量删除/向量库同步/全量重建索引 |
| RAG 智能问答 | 文本预处理/Embedding/向量检索/阈值判断/LLM生成/历史记录 |
| 工单管理 | 工单创建/查询/删除/状态流转/回访确认/一键入库 |
| 仪表盘 | 三角色统计视图 |
| 系统基础 | 健康检查/种子数据/CORS/异常处理 |

### 1.3 测试方法

- **功能测试**：手工 + 自动化（curl / Python requests），覆盖 36 个 API 端点
- **边界值测试**：空输入、超长输入、特殊字符、并发请求
- **回归测试**：修复后全量重跑核心用例
- **安全测试**：SQL 注入、XSS、未授权访问、越权操作、密码策略
