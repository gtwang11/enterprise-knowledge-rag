# 系统概述

## 运维数字员工门户

---

## 1. 课题来源

课题六《运维数字员工的建设》，选自《广东公司提供实训课题2026年版》。

课题原文要点：

> 为提升企业运营效率、为一线员工减负、为流程增效，企业采用 AI+RPA 等技术构建数字员工，通过软件机器人替代或辅助人类员工完成重复性高、规则性强或基于数据分析的复杂任务。

具体步骤：
1. 本地部署大模型 + RAG 私有知识库，构建运维常见问题 FAQ 问答对
2. 开发运维申告门户，具备账号管理、知识库管理、自助查询、工单流转功能
3. 后台运维人员定期处理机器难以处理的在线记录，进行人工干预和回访，处理完成后完善知识库

硬件要求：PC 机 4C8G 100GB 硬盘。测试环境需要连接互联网（首次下载模型）。

---

## 2. 业务场景

这是一个**企业内部 IT 运维支持系统**。核心就是让一线运维遇到问题先问 AI，AI 答不了再找人，人处理完以后把方案加回知识库，下次 AI 就能答了。

整个闭环：

```
一线运维人员（各地市基层工程师）
    │  日常遇到技术问题
    ▼
打开运维门户 → 输入问题 → AI 从知识库检索回答
    │                    │
    │                    ├── 能回答 → 返回答案 ✓
    │                    │
    │                    └── 不能回答 → 生成工单
    │                                      │
    ▼                                      ▼
查看"我的工单"状态              资深运维专家 领取工单
                                          │
                                          ▼
                                    人工研究后给出方案
                                          │
                                          ▼
                                    回访报障人，确认解决
                                          │
                                          ▼
                              一键录入知识库（下次 AI 就能回答了）

系统管理员：管理账号（入职/离职）、管理知识库内容质量
```

---

## 3. 三种角色

| 角色 | 谁 | 干什么 |
|------|-----|--------|
| 一线运维 (operator) | 基层运维工程师，在机房/基站/营业厅处理日常运维 | 遇到问题自助搜索答案；搜不到就提工单 |
| 资深专家 (expert) | 经验丰富的技术骨干/老师傅 | 处理 AI 搞不定的工单 → 回访确认 → 把方案录入知识库 |
| 管理员 (admin) | IT 部门管理者 | 管账号（入职开账号/离职冻结）、管 FAQ 知识库 |

---

## 4. 约束条件

| # | 约束 | 说明 |
|---|------|------|
| 1 | 不做手机 APP | 仅 Web 门户 |
| 2 | 不用 Docker | 直接命令行启动（start.bat / start.sh） |
| 3 | 单机部署 | 4C8G 100GB 一台 PC |
| 4 | 本地模型 | 大模型本地运行，不调云端 API |
| 5 | 三类角色 | operator / expert / admin，没有其他 |
| 6 | 全中文 | 界面和知识库都是中文 |

---

## 5. 功能模块一览

| 模块 | 谁用 | 干什么 |
|------|------|--------|
| 登录认证 | 所有人 | 账号密码登录、JWT 鉴权、密码修改、登录锁定 |
| 智能问答 | 一线运维 | 输入问题 → AI 检索知识库 → 回答/引导工单 |
| 我的工单 | 一线运维 | 查看自己提的工单、确认回访 |
| 工单处理 | 资深专家 | 查看待处理工单、领取、提交方案、驳回、一键入库 |
| FAQ 管理 | 管理员 | 增删改查 FAQ、Excel/CSV 批量导入、重建向量索引 |
| 账号管理 | 管理员 | 创建/编辑/冻结/解冻/重置密码 |
| 仪表盘 | 所有人 | 不同角色看到不同的统计面板 |

## 6. 技术架构（简版）

```
浏览器 (Vue 3 + Element Plus)
    │  REST API (JSON) + JWT
    ▼
FastAPI 后端
    ├── RAG 引擎 (jieba → Ollama embedding → ChromaDB 检索 → LLM 生成)
    ├── 工单状态机
    ├── 账号管理
    └── FAQ 管理
    │
    ├── SQLite (业务数据)
    ├── ChromaDB (768 维向量)
    └── Ollama (本地 11434 端口)
         ├── deepseek-llm-7b-chat:q4  (推理，~4.5GB)
         └── nomic-embed-text         (向量化，~0.3GB)
```

完整架构见 [SAD.md](./SAD.md)。
