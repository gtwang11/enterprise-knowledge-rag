# 部署与分支发布说明

本文说明如何把本项目推送到 GitHub 分支，以及如何让网页长期可访问，而不是每次手动在终端启动前后端。

## 1. 分支上传

推荐只提交源码、知识库生成脚本和知识库 JSON，不提交运行日志、虚拟环境、node_modules、SQLite/Chroma 运行目录。

本次新增/关键文件：

- `backend/scripts/build_ops_faq_kb.py`：生成官方/云厂商 FAQ、整合旧 FAQ、导入数据库、重建向量。
- `knowledge_base/official_ops_faq_1000.json`：官方/云厂商来源 FAQ。
- `knowledge_base/merged_ops_faq_2000.json`：与旧 `FAQ_1000条.json` 合并去重后的导入文件。
- `backend/engine/vector_store.py`：Chroma 检索失败时自动切换 SQLite 向量兜底检索。
- `backend/config.py`：本地 Qwen 生成超时时间调整为 120 秒。

建议分支名：

```bash
git checkout -b deploy/ops-knowledge-rag
git add backend/config.py backend/engine/vector_store.py backend/scripts/build_ops_faq_kb.py knowledge_base docs/部署与分支发布说明.md
git add backend/routers/qa.py backend/routers/ticket.py frontend/src/layouts/Sidebar.vue frontend/src/router/index.ts frontend/src/views/tickets/TicketDetailView.vue
git commit -m "build operations faq knowledge base"
git push -u origin deploy/ops-knowledge-rag
```

## 2. 为什么 GitHub 分支不能直接保证网页随时可访问

GitHub 分支只是代码仓库，不会自动运行 FastAPI、Ollama、Qwen 模型、SQLite 数据库和向量检索。

要让网页长期可访问，需要一台长期在线的机器，例如：

- 学校/组员已有服务器：如 `125.216.246.101`。
- 云服务器：阿里云、腾讯云、华为云等。
- 一台始终开机的本地电脑。

这台机器上需要常驻运行：

- Ollama 服务。
- `qwen2.5:7b` 模型。
- `nomic-embed-text` 模型。
- FastAPI 后端服务。

前端不建议长期使用 `npm run dev` 的 Vite 开发服务器。生产/演示时应执行 `npm run build`，由 FastAPI 托管 `frontend/dist`。

## 3. 服务器初始化流程

在服务器上拉取分支后执行：

```bash
git clone https://github.com/Hshhhhhhhhh/-enterprise-knowledge-rag.git
cd -enterprise-knowledge-rag
git checkout deploy/ops-knowledge-rag

python -m venv venv
source venv/bin/activate
pip install -r backend/requirements.txt

cd frontend
npm ci
npm run build
cd ..

ollama pull qwen2.5:7b
ollama pull nomic-embed-text

python backend/scripts/build_ops_faq_kb.py --import-db --reindex
```

如果服务器是 Windows，把 `source venv/bin/activate` 换成：

```bat
venv\Scripts\activate.bat
```

## 4. Linux systemd 常驻服务

如果服务器是 Linux，推荐用 systemd。假设项目在 `/opt/enterprise-knowledge-rag`：

```ini
[Unit]
Description=Enterprise Knowledge RAG Backend
After=network.target ollama.service

[Service]
WorkingDirectory=/opt/enterprise-knowledge-rag/backend
ExecStart=/opt/enterprise-knowledge-rag/venv/bin/python -m uvicorn main:app --host 0.0.0.0 --port 5173
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

保存为：

```bash
sudo nano /etc/systemd/system/enterprise-knowledge-rag.service
```

启用并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now ollama
sudo systemctl enable --now enterprise-knowledge-rag
sudo systemctl status enterprise-knowledge-rag
```

这样访问：

```text
http://服务器IP:5173
```

如果要用 80 端口或域名，建议再配 Nginx 反向代理。

## 5. Windows 常驻服务

如果部署在 Windows，推荐使用 NSSM 把后端注册为服务。

示例：

```bat
nssm install enterprise-knowledge-rag C:\path\to\repo\venv\Scripts\python.exe
nssm set enterprise-knowledge-rag AppDirectory C:\path\to\repo\backend
nssm set enterprise-knowledge-rag AppParameters -m uvicorn main:app --host 0.0.0.0 --port 5173
nssm set enterprise-knowledge-rag Start SERVICE_AUTO_START
nssm start enterprise-knowledge-rag
```

Ollama 也需要开机自启。Windows 上可以使用 Ollama 桌面端自启，或者同样用 NSSM/计划任务启动：

```bat
ollama serve
```

## 6. 更新代码后的部署流程

以后本地修改后推送到同一分支，服务器上执行：

```bash
git pull
source venv/bin/activate
pip install -r backend/requirements.txt
cd frontend && npm ci && npm run build && cd ..
python backend/scripts/build_ops_faq_kb.py --import-db --reindex
sudo systemctl restart enterprise-knowledge-rag
```

如果只是改前端或后端代码，没有改知识库，可以跳过 `build_ops_faq_kb.py`。
