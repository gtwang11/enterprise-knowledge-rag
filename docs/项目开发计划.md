## 1. 项目范围

做一个运维数字员工门户，功能清单：

1. 用户登录认证（登录/登出/密码修改/Token刷新/锁定限流）
2. 账号管理（创建/编辑/删除/冻结解冻/重置密码）
3. RAG 智能问答（问 AI → 答/答不了 → 建工单）
4. FAQ 知识库管理（增删改查、批量导入、向量库同步）
5. 工单管理（创建/领取/提方案/驳回/回访/一键入库）
6. 仪表盘（三个角色的统计看板）

不做 APP，不做 Docker，不调云端 API。单机部署。

## 4. 时间计划

实际做了一天，大致的顺序是：

| 时间段 | 做的事情 |
|--------|---------|
| 上午 | 项目初始化（创建项目结构、建表、配 Ollama、写 RAG 引擎核心代码） |
| 中午 | FAQ 管理 + 问答模块（CRUD 接口、向量同步、问答接口联调） |
| 下午 | 工单状态机 + 账号管理 + 前端页面 |
| 傍晚 | 仪表盘 + 中间件 + 全量测试 |
| 晚上 | 修 bug（15 个） + 写文档 |

## 6. 风险管理

| 风险 | 怎么应对 |
|------|---------|
| Ollama 模型下载慢 | 开发时提前下载好，不要等到部署时再下 |
| ChromaDB API 兼容问题 | 锁版本，写代码时先查文档确认接口没变 |
| LLM 生成太慢 | 设 30 秒超时，超了就走降级逻辑 |
| 时间不够 | 先保核心链路（问答 + 工单），其他功能能砍就砍 |

实际上做下来还算顺利。最大的时间花在修 ChromaDB 版本兼容问题上了——1.5.x 和 0.x 差得比想象的多。
